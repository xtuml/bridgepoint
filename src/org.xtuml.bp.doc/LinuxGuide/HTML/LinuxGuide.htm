1.    Introduction

BridgePoint supports editing, executing, and translating application models on both Windows and Linux operating systems.  Editing and executing models is handled natively on each OS by the java-based eclipse framework BridgePoint is built upon.  Translating models is handled by a Windows-based code generation application.  Running model translation on Linux requires additional software that is freely available from your Linux distribution vendor.

2.    Wine

Generator runs natively on the various Windows versions.  On Linux, Generator uses the Windows emulator Wine (http://www.winehq.com).  Wine is a freely available software package that is available for all Linux distributions.  Your system administrator must install Wine on your system if it is not already available.  To do this, run:

 

$ sudo yum install wine

 

Once Wine is set up, you can proceed with building and translating your model project inside BridgePoint.

3.    Translating Projects to Target Code

To translate and build the project, use the standard steps:

1.     Switch to the C/C++ perspective

2.     Highlight the project in the Resource Navigator view

3.     Choose Project > Build Project

 

 

The Wine library will cause two errors to be output during the code generation phase.  These errors can be safely ignored.  They are:

 

ALSA lib seq_hw.c:457:(snd_seq_hw_open) open /dev/snd/seq failed: No such file or directory


   fixme:imagehlp:CheckSumMappedFile (0x45b0000, 1787264, 0x1d5c7e4, 0x1d5c7dc): stub

4.    Additional Notes

Because of the variances among the myriad of Linux distributions (even from a single vendor), your sysadmin may need to make additional configuration changes to allow the build to work smoothly.  Known items are captured here in hopes they will be useful to BridgePoint users.

Customized Linux Environments

Due to the myriad of Linux distributions and library extensions available, the environment that BridgePoint is installed into may be insufficient for correct operation. As a result, your sysadmin may need to make additional configuration changes to allow BridgePoint to work smoothly within your unique setup.  At this time, BridgePoint requires the following packages in addition to the core Linux distribution files:

 

compat-libstdc++-33-3.2.3-69

gtk2-2.18.9-6

java-1.6.0-openjdk-1.6.0.0-1.41.1.10.4

wine-1.2.3-1

gcc-c++-4.4.6-3

 

These packages also have dependencies, which depending on your existing environment, can lead to further downloads. Note, that Wine is needed for code generation.

Using Unsupported Linux Variants

BridgePoint is officially supported on RHEL5.  However, it can be run on other Linux distributions. To run BridgePoint model translation on Ubuntu (a debian-based distribution), your system administrator must install libstdc++5.  To do this, run:

 

$ sudo apt-get install libstdc++5

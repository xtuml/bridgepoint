Overview

BridgePoint Builder allows you to model an application in xtUML that is independent of it's implementation. The models capture all of the data (classes and associations), control (state machines), and processing (actions described in OAL) in an implementation-independent manner.

Whereas the metamodel describes the data and control as classes and associations, the actions are described solely within class attributes. The data and the control of the system are translated in one manner, through the specification of what is termed structural rules . The processing in the system is translated in another manner, through the specification of action translation rules .

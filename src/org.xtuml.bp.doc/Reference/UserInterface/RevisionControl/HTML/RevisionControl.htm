Revision Control

BridgePoint model files are version controllable in either the Team CVS interface or the Team ClearCase interface in Eclipse. The following sections discuss these interfaces.

Using the Team CVS Interface

To access the Team CVS interface, click Window > Open Perspective > Other… > CVS Repository Exploring. When you access this perspective for the first time, it will not contain any repositories as shown below.

 

To add a repository to this perspective, perform the following steps:

1.     Right-click inside the perspective and click New > Repository Location… in the context menu. The Add CVS Repository dialog box appears.

2.     Enter the appropriate host and repository path locations. Then, enter your CVS user name and password.  You will need to obtain this information from your CVS administrator. 

3.     Click Finish.

Now, you have a repository location listed in the CVS Repositories perspective.

Please refer to the Team CVS help documentation located in the Eclipse help under Workbench User Guide > Getting Started for more information on using the CVS repository in Eclipse.

Using the Team ClearCase Interface

Please click Getting Started > Rational ClearCase SCM Adaptor in the Eclipse help for instructions on activating the Team ClearCase interface and on setting your preferences.

The concept of read-only files in BridgePoint is important for the proper use of the Team ClearCase interface. It is important for you to recognize when a file is read-only. Read-only models are not modifiable in BridgePoint. In order to determine if a model is marked as read-only you can look for certain characteristics. The characteristics of a read-only model are listed below.

§  All menu options which can lead to model changes are disabled.

§  Models or model elements can not be renamed.

§  Model element properties can not be modified.

§  Changes to diagrams are ignored.

§  All symbol drawing tool buttons are disabled.

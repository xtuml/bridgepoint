Calculator Domain Step-by-Step

Overview

Shlaer-Mellor models are composed of domains which encapsulate classes arranged in class diagrams. A class may have a state machine which sequences the dynamic lifecycle of the class. Domain functions provide synchronous services to be invoked by other communicating domains. Action language defines the processing which occurs in domain functions, state actions and class operations.

The following tutorial walks a new user through the steps of creating a Shlaer-Mellor model in BridgePoint. BridgePoint uses UML notation to represent Shlaer-Mellor elements. A UML Component represents a domain. Inter-domain communication is modeled through UML Interfaces. Terminators are modeled as Ports containing Operations which serve to represent terminator services.

The domain being modeled is an abstract arithmetic logic unit (ALU) of a simple hand-held calculator.

The tutorial will follow a logical sequence. Note that it is fine to define the functions and interfaces before the class diagram and state machines.

  1. xtUML Project

  2. Domain Component

  3. Class Diagram

  4. State Machines

  5. Domain Functions

  6. Interfaces

Tip
 In the sections below, click on highlighted text to see screenshots which provide additional clarity.

xtUML Project

  1. Create an Eclipse xtUML Project with the name of the new domain ('ALU' in this case) by right-clicking in the Model Explorer view (in empty space) and selecting New > xtUML Project.

  2. Click Next to set the model compiler to WASL Exporter.

Domain Component

  1. Create a top-level package (with the same name as the domain) by right-clicking on the project in the Model Explorer view and selecting New > Package.

  2. Inside the top-level domain package, create a component with the same name. This can be done by right-clicking on the ALU package and selecting New > Components > Component. Or, after opening the Graphical Editor (canvas) of the ALU package by double-clicking on it and then by clicking on Component in the Components drawer of the Palette and drawing the component on the editor canvas (by clicking and dragging the mouse cursor) of the package.

Class Diagram

  1. Inside the domain component, create a package called 'classes'. Do this by right-clicking on the ALU component and New > Package and give the package the name 'classes'.

  2. Double-click on the classes package to open the class diagram canvas.

  3. Inside the classes package, create classes 'Calculator', 'Display', 'Memory', and 'Operation' by clicking on Class in the Classes drawer of the Palette and then drawing each class on the canvas.

  4. After you create each class, select the class (by clicking on it) to set the class key-letters in the Properties view to 'CALC', 'DISP', 'MEM' and 'OP' respectively.

Note
 The class key letters default to be the same as the class name.

Move a class around by clicking and dragging it. Arrange the class diagram sensibly in anticipation of the associations to be added between related classes.

Resize the classes by clicking on the class to select it and then clicking and dragging a corner or a side.

Attributes

  1. In the Model Explorer right click on the Calculator class and New > Attribute. Name the attribute 'calc_id'.

  2. Right click on the calc_id attribute and Add to Identifier…​ and add this attribute to the primary identifier (*1).

  3. Add another attribute called 'current_op'.

  4. Right click on the Memory class and add an attribute called 'value'.

  5. Right click on the value attribute and Set Type…​. In the Type Selection window choose real as the data type for this attribute.

Add other attributes and practice adding and removing attributes from identifiers. Create an attribute and set up a secondary identifier.

Associations

  1. Establish a one-to-one association between the Calculator and Display classes by clicking on the Classes drawer in the Palette and then clicking on the Association widget.

    1. Notice the cursor turn into a little "plug" when you move it onto the canvas.

    2. Click inside the Calculator class and drag to inside the Display class and then "let go".

    3. Fill out the Edit association edit form.

      1. Make the Association Number '1'.

      2. Edit Calculator Unconditionally prints_answer_on One Display.

      3. Edit Display Unconditionally displays_value_for One Calculator.

      4. Leave the Association check box empty.

      5. Change Use identifier from to select class Calculator.

      6. Finish.

    4. If the association is drawn as a diagonal straight line ("oblique routing"), this can be changed by right-clicking on the association and selecting Routing > Rectilinear Routing.

    5. In the Model Explorer view under the Display class, right click on the newly added calc_id referential attribute and Add to Identifier…​ and make this attribute the primary identifier (*1).

  2. Establish a 1:1 unconditional association between the Calculator and Memory classes such that:

    1. Calculator Unconditionally stores_value_for_calculation_in One Memory.

    2. Memory Unconditionally stores_value_for One Calculator.

    3. Use identifier from Calculator.

    4. In the Model Explorer view under the Memory class, add the newly formed calc_id referential attribute to the primary identifier.

  3. Establish an associative relationship between the Memory and Display classes using the Operation class as the associative class.

    1. Select the Association widget from within the Classes drawer of the Palette.

    2. Draw the association from the Memory class to the Display class.

    3. Supply (symmetric) role phrase 'combines_with' to both role phrase boxes.

    4. Check the Associative box and select Operation as the Associative class.

    5. Check the Formalized box and select both primary identifiers.

    6. Finish.

    7. Combine the duplicate referential attributes in the Operation class by right-clicking on the first calc_id and Combine With…​ and then select the other calc_id attribute in the Combine With chooser.

Association lines can be moved and bent using the mouse and clicking on end-points or bend-points.

Role phrases can be repositioned by dragging them with the mouse.

Operations

  1. Add an operation called 'init' to the Calculator class by right clicking on the Calculator class in the Model Explorer view or by right-clicking on the Calculator class on the canvas and New > Operation.

  2. Add an operation called 'store' to the Memory class.

    1. Add a parameter to the store operation by right-clicking on the store operation in the Model Explorer view and New > Parameter and giving it the name 'value'.

      1. Set the type of the value parameter by right-clicking on it and Set Type…​ and choosing real.

  3. Add an operation called 'evaluate' to the Operation class.

  4. Add an operation called 'clearDisplay' to the Display class.

  5. Add action language to the operations by double-clicking on the operations in the Model Explorer view. An action language editor will be opened.

State Machines

  1. Add a state machine to the Display class by right-clicking on the class and New > Instance State Machine.

  2. Double-click on the Display class to edit the newly created state machine.

  3. Inside the Display state machine, create a state by clicking on State in the Palette and drawing a state on the canvas. Give it the name 'displaying_memory_value'.

  4. Add another state called 'toggling_sign' in the same way.

  5. Add an event to the Display state machine by right clicking on the state machine canvas or by right-clicking on Instance State Machine under the Display class in the Model Explorer view. Then New > Event. Name the event 'keypress'.

  6. Add a transition from the displaying_memory_value state to the toggling_sign state.

    1. Select Transition from the Palette.

    2. Drag the mouse cursor from inside the displaying_memory_value state to the inside of the toggling_sign state.

    3. Right-click on the newly added transition and Assign Event…​. Choose the keypress event from the selection box.

    4. Arrange and bend the transition and adjust the position of the event as desired.

  7. Add action language to the states by double-clicking on the states on the canvas or in the Model Explorer view. An action language editor will be opened.

Domain Functions

  1. Inside the domain component, create a package called 'functions'. Do this by right-clicking on the ALU component and New > Package and give the package the name 'functions'.

  2. Inside the 'functions' package, New > Function to add domain functions 'clear' and 'key'.

  3. Right click on the function 'key' and New > Parameter to add parameter 'code'.

Scenarios

  1. Inside the domain component, create a package called 'scenarios' the same way that you created the 'functions' package.

  2. Inside the scenarios package, New > Function to add initialization function 'init'.

Interfaces

Interfaces define the signatures for all inbound messages (domain services) and outbound messages (terminator services). Data types of parameters on messages must be shared between domains. Thus, these shared data types are defined outside the component together with the interfaces.

  1. Inside the top-level domain package (ALU), create a package 'Shared'. This package will be a peer to the domain component 'ALU'.

Shared Data Types

  1. Inside the Shared package, New > Types > User DataType. Name the new type 'keyscan'.

    1. Be sure that keyscan is based on type integer. This can be confirmed in the Properties view when the keyscan type is selected.

Terminator Interfaces

  1. Add an interface called 'disp' by right-clicking on the Shared package and New > Components > Interface.

    1. Add an operation to the disp interface by right-clicking on disp and New > Operation. Give the operation a name of 'result'.

      1. Ensure the result Message Direction is ToProvider when the result parameter is selected. This can be confirmed in the Properties view when the keyscan type is selected.

      2. Add New > Parameter 'value' to the result operation.

      3. Set Type…​ of parameter 'value' to real.

    2. Add Operation 'error'.

      1. Ensure error Message Direction is ToProvider.

      2. Add parameter 'message'.

      3. Set Type…​ of message to string. (Note that you also need to set the By Reference Indicator to By Value (default) for input parameters and By Reference for output parameters.)

Domain Service Interface

  1. Add an interface called 'ALU' in the same way that you added other interfaces to the Shared package.

  2. Publish domain functions into the ALU interface.

    1. In the Model Explorer view, navigate to the functions package containing the domain functions defined for this component.

    2. Multi-select the key and clear functions by clicking on one and the holding the shift key while clicking on the other.

    3. Right-click and Publish To Interface…​ and click OK.

    4. Choose the 'ALU' interface in the Publish To Interface Selection window.

Terminator and Domain Ports

  1. Create ports for the terminator and domain interfaces by opening the top-level domain package and adding them.

    1. Double-click on the top-level ALU package so that you can see the ALU component on the canvas editor view. Resize and rearrange as needed.

    2. Provide the domain services port.

      1. In the Palette select Provided Interface from within the Components drawer.

      2. Click and drag from inside the ALU Component to just outside the component boundary.

      3. Right-click on the newly added port (interface reference) and Formalize. Select the ALU interface from the Interface chooser.

    3. To provide the terminator services port.

      1. In the Palette select Required Interface from within the Components drawer.

      2. Click and drag from inside the ALU Component to just outside the component bounary.

      3. Right-click on the newly added port and Formalize. Select the disp interface from the Interface chooser.

    4. Rename the ports from their default names, 'Port1' and 'Port2', to match the domain names ('ALU' and 'disp'). Set the *key letters field in the Properties view.

Generate WASL

  1. There are two ways to generate WASL from the graphical user interface. (There are other ways using a command line interface (CLI).)

    1. Right-click on the Eclipse xtUML Project and Build Project. See the export log scroll in the Console view.

    2. Click on the Eclipse xtUML Project to be sure it is selected. Then click the Hammer icon in the Tool Bar. See the export log scroll in the Console view.

    3. Open the Resource Perspective (via the widget indicated) or via Window > Perspective > Open Perspective > Other…​ > Resource, and navigate to the /wasl folder of the project to browse the generated WASL.

Tip
 Navigate between each open Perspective by clicking on the icon in the upper right-hand corner.

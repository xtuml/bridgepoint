13.5. Assignability of Names

Names are used to refer to declared entities. A declared entity is an attribute, a parameter or a variable. Names can also denote components, elements and attributes of other names, or attributes of the instance denoted by the reserved word this.

[185] name = attribute name | parameter name | variable name | indexed component | selected component | selected attribute | slice ;  

A compile-time error occurs if:

  • An attribute name does not name an attribute of the enclosing object. Note that this will only be possible within an instance state or an instance service.

  • A parameter name does not name a parameter of the enclosing state or service.

  • A variable name does not name a variable currently in scope.

  • this is used outside an instance state or an instance service.

Certain forms of name ( indexed components, selected components and selected attributes ) include a prefix that designates some related entity.

The evaluation of a name determines the entity denoted by the name. This evaluation has no other effect for a name that is an attribute name, a parameter name or a variable name.

The evaluation of a name that has a prefix includes evaluation of the prefix. The evaluation of the prefix consists of the evaluation of the name or the reserved word this. The type of the prefix is the type of the name or reserved word this.

A name may be used as an expression (see Section 14.4.3, “Names as Expressions”).

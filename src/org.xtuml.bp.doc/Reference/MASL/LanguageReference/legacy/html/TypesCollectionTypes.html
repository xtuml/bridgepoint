A collection type is a type whose values consist of zero or more elements all of which have the same type. Four forms of collection types are provided; sets, bags, sequences and arrays.

[45] collection type = set type | bag type | sequence type | array type ;  

A set is a mathematical set. It does not contain duplicate elements. A bag is a like a set, which may contain duplicates. A sequence is like a bag in which elements are ordered. Both bags and sets have no order defined on them. An array is like a sequence except that the number of elements in the array is fixed at compile-time.

The features of the each of the collection type is given in Table 3.4, “Collection Type Features”.

Table 3.4. Collection Type Features
Collection Type Maintains Insertion Order Allows Duplicates Fixed Length
set No No No
bag No Yes No
sequence Yes Yes No
array Yes Yes Yes


3.3.1. Set Types

A set type is a collection type whose values do not contain duplicate elements and which have no order defined on them.

[46] set type = 'set of', type ;  

The features of a set type are given in Table 3.5, “Set Type Features”, where s and s2 represent entities of the type on which the feature is applied.

The features ordered_by and reverse_ordered_by are only defined for set types whose element type is an instance type.

The feature -> is only defined for set types whose element type is an instance type.

Table 3.5. Set Type Features
Feature Description
union Returns the union of s and s2. The result set contains all the elements in either set.
intersection Returns the intersection of s and s2. The result set contains all the elements common to both sets.
not_in Returns the set containing all the elements in s that are not in s2.
disunion Returns the set containing all the elements in either s or s2, but not in both.
& Returns the set containing all the elements in s and s2.
ordered_by Returns a sequence that contains all the elements of s in specified ascending order.
reverse_ordered_by Returns a sequence that contains all the elements of s in specified descending order.
find Returns a set containing all the elements of s that satisfies the specified condition.
find_all Returns a set containing all the elements of s.
find_one Returns an arbitrary element of s that satisfies the specified condition.
find_only Returns the only element of s that satisfies the specified condition.
-> Returns the bag of elements that are related to the elements of s using the specified relationship.


3.3.2. Bag Types

A bag type is a collection type whose values can contain duplicate elements which have no order defined on them.

[47] bag type = 'bag of', type ;  

The features of a bag type are give in Table 3.6, “Bag Type Features”, where b and b2 represent entities of the type on which the feature is applied.

The features ordered_by and reverse_ordered_by are only defined for bag types whose element type is either a structure or an instance type.

Table 3.6. Bag Type Features
Feature Description
union Returns the union of b and b2. The result set contains all the elements in either bag.
intersection

Returns the intersection of b and b2. The result set contains all the elements that are in both bags.

not_in Returns the bag containing all the elements in b not in b2.
disunion Returns the bag containing all the elements in either b or b2, but not in both.
& Returns the bag containing all the elements in b and b2.
ordered_by Returns a sequence that contains all the elements of b in specified ascending order.
reverse_ordered_by Returns a sequence that contains all the elements of b in specified descending order.
find Returns a bag that contains all the elements of b that satisfy the specified condition.
find_all Returns a bag that contains all the elements of b.
find_one Returns an arbitrary element of b that satisfies the specified condition.
find_only Returns the only element of b that satisfies the specified condition.
-> Returns the bag of elements that are related to the elements of b using the specified relationship.
get_unique Returns a set containing all the elements of b.


The features find, find_all, find_one and find_only are only defined for bag types whose element type is an instance type.

The feature -> is only defined for bag types whose element type is an instance type.

3.3.3. Sequence Types

A sequence type is a collection type whose value can contain duplicate elements which have order.

[48] sequence type = 'sequence', ['(', expression, ')'], 'of', type ;  

Sequences are indexed by values of the pre-defined type positive which is declared in the standard domain.

The features of a sequence type are given in Table 3.7, “Sequence Type Features”, where s and s2 represent entities of the type on which the feature is applied.

The features ordered_by and reserve_ordered_by are only defined for sequence types whose element type is either a structure or an instance type.

The features find, find_all, find_one and find_only are only defined for sequence types whose element is an instance type.

The feature -> is only defined for sequence types whose element is an instance type.

The optional expression defines the upper bound of the sequence and must equate to a positive qualified type.

Table 3.7. Sequence Type Features
Feature Description
& Returns the set containing all the elements in q and q2.
[] Returns the either the element of q at a specific index or the slice of q given by a range of indices.
ordered_by Returns a sequence that contains all the elements of s in specified ascending order.
reverse_ordered_by Returns a sequence that contains all the elements of s in specified descending order.
find Returns a bag that contains all the elements of q that satisfy the specified condition.
find_all Returns a bag that contains all the elements of q.
find_one Returns an arbitrary element of q that satisfies the specified condition.
find_only Returns the only element of q that satisfies the specified condition.
-> Returns the bag of elements that are related to the elements of q using the specified relationship.


3.3.4. Array Types

An array type is a collection type whose values can contain duplicate elements which have no order and has a fixed number of elements.

[49] array type = 'array', '(', range, ')', 'of', type ;  

Arrays are indexed by values given by the type of its range. This range must resolve to some specific integer or enumeration type. If the type of the range resolves to numeric type, then the range is defined to be of the pre-defined type integer with bounds given by a conversion to integer of the bounds of the range.

The features of an array type are given in Table 3.8, “Array Type Features”, where a represents the instance on which the feature type is applied.

The features ordered_by and reserve_ordered_by are only defined for array types whose element type is either a structure or an instance type.

The features find, find_all, find_one and find_only are only defined for array types whose element is an instance type.

The feature -> is only defined for array types whose element is an instance type.

Table 3.8. Array Type Features
Feature Description
[] Returns the either the element of a at a specific index or the slice of a given by a range of indices.
ordered_by Returns a sequence that contains all the elements of a in specified ascending order.
reverse_ordered_by Returns a sequence that contains all the elements of a in specified descending order.
find Returns a set that contains all the elements of a that satisfy the specified condition.
find_all Returns a set that contains all the elements of a.
find_one Returns an arbitrary element of a that satisfies the specified condition.
find_only Returns the only element of a that satisfies the specified condition.
-> Returns the sequence of elements that are related to the elements of a using the specified relationship.
get_unique Returns a set containing all the elements of the array.

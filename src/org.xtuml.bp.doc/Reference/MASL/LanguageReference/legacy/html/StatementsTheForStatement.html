12.14. The For Statement

12.14.1. Abrupt Completion

The for statement executes a loop parameter specification, then executes a sequence of statements repeatedly until all the values specified by the loop parameter specification have been covered.

[172] for statement = 'for', loop parameter specification, 'loop', sequence of statements, 'end', ['loop'], ';' ;  
[173] loop parameter specification = variable name, 'in', ['reverse'], range ;  

For execution of a for statement, the loop parameter specification is first elaborated. This elaboration creates the loop parameter and elaborates the range. If the range defines a empty range, then execution is complete. Otherwise, the sequence of statements is executed once for each value of the range (or until the for statement is left as a consequence of a transfer of control). Prior to each iteration, the corresponding value of the range is assigned to the loop parameter. These values are assigned in increasing order unless the reserved word reverse is present, in which case the values are assigned in decreasing order.

The range shall resolve to some specific integer or enumeration type. If the type of the range resolves to a numeric, then the range is defined to be of the pre-defined type integer with bounds given by a conversion to integer of bounds of the range.

12.14.1. Abrupt Completion

Abrupt completion of the contained sequence of statements is handled in the following manner:

  • If execution of the sequence of statements completes abruptly because of an exit statement, no further action is taken and the for statement completes abruptly for the same reason.

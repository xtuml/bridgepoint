3.    General Notes

  • A summary of the release highlights and new functionality is available on the Whats New page.
  • In this release BridgePoint has been modified to pass bridge parameters in the user-defined order when executing realized bridge code. This may cause unexpected failures in Java realized code as the values passed may not match the modeled external entity. To address this issue there are two approaches, one where the Java code is modified and the other where the modeled external entity is modified. In any case both must have a matching parameter ordering.
  • The provided and required interface tools can now be used to draw a connection between two components in a single stroke, functioning as an assembly connector.  The connection created will produce a provided interface and a required interface.  This allows for a simpler approach to creating interface satisfactions.
  • xtUML eXecute now creates dynamic arrays with an initial size of 0.  Previously they were created with an initial size of 1.  Models that relied on the initial size being 1 must be updated to account for the new behavior.
  • When using Java Realized Components with JNI, DLLs are not unloaded when the session terminates.  After terminating the session, the option to "Clear Terminated Sessions" (the X icon) must be run to clear these sessions.  The terminated sessions hold resources that prevent the DLL from unloading.  Once those terminated sessions are cleared, the DLL is unloaded.
  • IMPORTANT: The legacy plug-in com.mentor.nucleus.bp.mc.mc3020 has been removed.  A utility is provided that allows users to “Set Model Compiler”.  Users must run this tool on any projects that reference the old builder before xtUML Translate will run.  To run the tool, select a project, right-click to bring up the context menu, then choose “BridgePoint Utilities > Set Model Compiler”.   See BridgePoint UML Suite Help > Reference > Translation Reference > Setting the Model Compiler for full details.
  • IMPORTANT: BridgePoint 4.0.0 and later releases represent a major change in the licensing of the tool.  The xtUML Editor is now unrestricted.   The xtUML eXecute Application (formerly Model Verifier) and xtUML Translate (Model Compiler) tools remain licensed Mentor Graphics products.  Customers who wish to use this and later versions of BridgePoint must contact either their sales representative or customer support in order to transition their BridgePoint licenses to the new part numbers.
  • IMPORTANT: BridgePoint 4.0.0 and later releases require the version 11.11 of the FlexNET licensing software. Users of floating license servers are required to upgrade to this version or later.  The latest version of FlexNet is available here on Mentor Graphics SupportNet.

 

 

4.    Interoperability with Previous Releases

BridgePoint 4.1.7 is fully data compatible with workspaces and projects created with BridgePoint 3.7.0 and xtUML.org’s xtUML Editor 3.7.2.  BridgePoint 4.1.7 can open model projects created with BridgePoint 3.5.x and 3.6.x, but requires the projects to be upgraded completely to generic packages just like the 3.7.x series of tools.

 

 

5.    Known Issues

Installer

  • Cannot install from folder with unicode character in path 
  • Install into folder with unicode character in path not working. 

 

xtUML Editor

  • Port multiplicity is partially supported. It is present in the meta-model. The user interface allows setting One and Many (in properties). A future release of BridgePoint will allow integer values and will be displayable on the canvas.
  • Defects in states designated as 'final'. Final state is available for class-based state machines. Transitions can be drawn out of final states. States with out-going transitions can be marked as final.
  • Failure to obtain a license may result in entries being placed in the Error Log that look like runtime defects.
  • Setting the name prefix mode of a non-referential attribute to 'referred to prefix' causes the attribute to be designated as orphaned. Resetting the naming prefix mode to 'no prefix' returns the attribute name to its original value.
  • Setting a function's Successful Parse Indicator to "Do not parse" does not clear the function's parse errors from the Problems list.
  • If model execution or model compilation is to be performed on a modeling project modified using the xtUML.org Open Sourced editor, the project must be initially created using BridgePoint (licensed product).

 

Verifier

  • Using the return statement in a Mathematically Derived Attribute (MDA) is incompatible with the semantics of MDAs. The act of writing the attribute value is considered the end of the computation. Return statements should therefore not be used in MDA OAL, even though the parser permits it.
  • Pushing the suspend button during Verifier execution causes Verifier to stop what it is doing immediately. It stops whether it is processing an activity or not. If it should stop between activities, then the Debug View may show only events pending (or none at all if Verifier is waiting for an external stimulus). Verifier will only source locate (i.e. open an activity editor) if it stops inside of an activity; so in the cases outlined above, no source location will occur. If this happens and there are events in the queue, pushing the Step Into button will enter the next state action and source location will immediately occur.
  • Verifier does not fully support instance reference passing.

 

Code Generator Engine

  • Any of these fonts installed on a Windows XP may cause the generator to hang soon after it starts:
    • MS-DOS CP 437
    • MS-DOS CP 949
    • Adobe Ming
    • Adobe Myungjio
    • Adobe Kozuka
    • Adobe Ryo

The MingLiU/PMingLiU font (mingliu.ttc) may cause generator to hang as well.

Enabling East Asian font support on a Windows XP Professional computer usually installs the DOS fonts listed above. If you do install EA-font-support you may want to save off the fonts, remove EA-support and then copy the fonts back in. EA-support causes a big performance hit on American English computers.

  • The names of emit files may contain ASCII characters only. International characters such as umlauted characters are translated to 'Z' before the file is created and generator output is emitted to it.
  • Set manipulation error message. The generator reports errors when an empty instance is subtracted from an instance reference set.
  • Unbalanced tick marks are silently ignored by BridgePoint Generator. Unbalanced tick marks (apostrophes) within string constants are silently interpreted by the Generator as string delimiters.
  • BridgePoint Generator defect while parsing select-where on multiple conditions. If a select-where statement contains multiple conditions, there must be a space between 'where' and the opening parenthesis. Otherwise the parser fails to correctly parse the statement.
  • BridgePoint Generator does not compare select cardinality with schema.
  • BridgePoint Generator will not start on W2K if %USERNAME% contains spaces.

 

MC-3020

  • Events cannot be numbered with 0.
  • Some platforms have no real time clock. Therefore date and time may not be available in standard Gregorian or Caesarian form.

 

Eclipse CDT

  • Eclipse CDT has an internal error when attempting to rebuild an executable with spaces in the name.  The recommended workaround is to open the project properties, then navigate to C/C++ Build > Settings > Build Artifact.  In the Artifact name field, replace “${ProjName}” with an exact name with no spaces.

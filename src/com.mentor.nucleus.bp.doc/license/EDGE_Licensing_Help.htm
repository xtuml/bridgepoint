BridgePoint Licensing Help

The BridgePoint Tools do not work properly without a license key, even though the Workbench appears.

NOTE: You must restart BridgePoint after making changes to the license file.

Step 1: Retrieve Your License File

You received your license file by email. If you do not have this email, please contact your sales representative.

Step 2: Install Your License

Three types of licenses exist.  Follow the steps below corresponding to the type of license you have chosen for your site:

·         Node-locked license

·         Floating license

·         Dongle license

Node-Locked Licenses

If you have an Ethernet MAC address node-locked license:

1.     Check the host ID of your machine by issuing the following command at a command prompt:

<install_dir>\license\lmutil lmhostid

You should get a message similar to the following:

The FLEXlm host ID of this machine is "<hostnumber>"

The message informs you of the ID number of the host machine. If the host ID number does not match the ID listed in the license.dat file, then you will need to request a corrected license with a matching host ID.  Find the license.dat file in the license directory under your BridgePoint installation.  Typically, the location is C:\MentorGraphics\BridgePoint\license\license.dat .

2.     If the host ID of your machine matches the ID in the license.dat file, copy the license keys from the email to the license.dat file.  FLEXlm reads the license.dat file to confirm that your installed products are properly licensed.

The license keys are formatted similarly to the following example:

INCREMENT bpumlbc_c mgcld 2008.070 12-jul-2008 0 2D76CC7C2AADC2207F64 \

    VENDOR_STRING=E2AF11A3 HOSTID=001a722fbf81 SN=8742958 SIGN2="0870 6608 \

    7164 238D 9797 2112 9E8F CD41 7534 45F4 016C 5A9C E639 9DAC B0E7 0BFF \

    DC87 22B6 FF7F AB06 277D 0E37 8DE2 72A3 375F 16FE 306D C6B8 B1E6 9BD6"

INCREMENT bpmdlcmpc_c mgcld 2008.080 13-aug-2008 0 4D364E7D3A950157613C \

    VENDOR_STRING=70C4D8FD HOSTID=001a722fbf81 SN=8953356 SIGN2="030F FCA9 \

    B47F DE6D 281F 7E45 503C 6BE9 5CF4 6059 3115 7ADB BA34 D069 DF32 064C \

    12D3 4138 9C77 8B72 84E1 7F4D EFB2 822B 0BDB D789 94C9 1E2A 0499 8E4B"

The host ID is listed in the second line (for example, 0001a722fbf81). The INCREMENT lines indicate which components you are authorized to use:

·         bpumlbc_c -- BridgePoint UML Suite w/ Binary C & C++ Model Compilers

·         bpmdlcmpc_c -- BridgePoint UML Source C Model Compiler (MC-3020)

3.     The environment variable MGLS_LICENSE_FILE tells BridgePoint where to find license keys.  As of BridgePoint 2.1.0, this variable is no longer set by the installer in the system environment.  BridgePoint now sets this variable in an application-specific manner using the file Launcher.bat in the eclipse installation chosen at install time (typically C:\MentorGraphics\BridgePoint\eclipse). If MGLS_LICENSE_FILE is set in the system environment, it is used.  If it does not exist, the launcher defaults to C:\MentorGraphics\BridgePoint\license\license.dat .  If your license keys are stored somewhere other than the default location, you must edit the launcher batch file to point to that location.

4.     Restart BridgePoint.

You are now ready to run your installed Mentor Graphics products.

Floating Licenses

License Server Setup

To set up a floating license server running Windows:

1.     Execute the <CD-ROM>\license\SETUP.EXE in the BridgePoint Distribution CD Image on your server.  This will install the license server to a directory on your machine.  This install directory is hereafter called <license_manager_dir>.

2.     FLEXlm reads the license.dat file to confirm that your installed products are properly licensed.  On the designated server machine, import the license keys from the email to a text file called <license_manager_dir>\license.dat.

The license.dat file must contain the following:

·         A SERVER line to identify the machine you have designated as the license server.

·         One or more DAEMON lines to specify the name and location of your license vendor daemons.

·         One or more INCREMENT lines specifying the BridgePoint components you are licensed to use.

The following is an example of a license.dat file for a two-user BridgePoint floating license:

SERVER svr-inl-eng-01 0013110EB477 1717

DAEMON mgcld C:\MentorGraphics\Licensing

INCREMENT bpumlbc_c mgcld 2008.100 17-oct-2008 2 DD07D351F80BFF03100E \

    VENDOR_STRING=9C21CD55 SN=9524716 SIGN2="1B40 E1E1 C4A4 8938 BC6A 9915 \

    9133 373E F7FB EF1B E5A0 35E3 971C 0033 EFE2 12B6 2EA3 F4C4 EEC3 63A3 \

    18DE A120 4123 4E0B 8947 C5CA 0E07 D2AD 7C93 2859"

INCREMENT bpmdlcmpc_c mgcld 2008.100 17-oct-2008 2 1DA7C371E4749457FB63 \

    VENDOR_STRING=DCD8A5B1 SN=9524717 SIGN2="1F25 E4A2 C85F DA2F 9E69 AEAC \

    640F C966 F41F BA77 5D25 735F 0316 B3A1 5399 1F6C 259E 392C F662 56D2 \

    A829 CF98 87AB E440 0920 D4F3 7F36 0E1D EBFB E620"

3.     Verify that the information on the SERVER line is correct. The SERVER line is formatted as follows:

SERVER <server_name> <hostID> <port_number>

where server_name is the name of the server machine, hostID is the host ID number, and port_number is the unique TCP/IP port number assigned to your product installation.

Verify that the host ID of the server machine matches the hostID entry on the SERVER line by issuing the following command at a command prompt:

<license_manager_dir>\lmutil lmhostid

You should get a message similar to the following:

The FLEXlm host ID of this machine is "<hostnumber>"

The message informs you of the ID number of the host machine. If the host ID number does not match the ID listed in the license.dat file, you must request a corrected license with a matching host ID.

4.     Modify the DAEMON line(s) in the license.dat file to point to the location of your license daemon. In the example above, the path to the license daemon is C:\MentorGraphics\Licensing.

5.     Select Start > Settings > Control Panel > System.  In the System Properties dialog box, select the Advanced tab, then the Environment Variables button. In the System variables list, verify that the MGLS_LICENSE_FILE environment variable is pointing to the location of your license.dat file. The FLEXlm license manager uses the MGLS_LICENSE_FILE variable to search for your license file.  Correct this path if it is incorrect.

6.     On your server, run the <license_manager_dir>\lmtools.exe license configuration utility.

a.     Under the Service/License File tab, select Configuration using Services.

b.    Click the Config Services tab. In the Service Name pull down menu type “BridgePoint license server” or use the default name of “FLEXlm Service 1”.  Next, specify the full path to the following files:

Executable file -- lmgrd.exe

License file -- license.dat

Debug log file -- logfile.log.

c.     To start running the server software automatically whenever the server powers up, click the Use Services checkbox, and then click the Start Server at Power Up check box.

d.    Click Save Service.  When prompted, confirm that you want to save the FLEXlm Service 1 service.

e.     Click the Server Status tab.  Click Perform Status Enquiry.  It should indicate the service is "UP" and list information about various feature codes starting with "xtuml..."

f.     Click the Config Services tab. Click View Log and verify the log file does not indicate any problems.

NOTES:

·         Use the buttons on the Start/Stop/Reread tab to manage the state of your named service.

·         In some installations, the server must be rebooted to launch the license service successfully. 

·         If you are using Windows XP SP2 on your server machine the default firewall settings prevent clients from checking out licenses. You need to add lmgrd and any vendor demons (mgcld) to the Firewall Exceptions List.

Application Client Setup

The environment variable MGLS_LICENSE_FILE tells BridgePoint where to find license keys.  As of BridgePoint 2.1.0, this variable is no longer set by the installer in the system environment.  BridgePoint now sets this variable in an application-specific manner using the file BridgePoint_Launcher.bat in the eclipse installation chosen at install time (typically C:\MentorGraphics\BridgePoint\eclipse). If MGLS_LICENSE_FILE is set in the system environment, it is used.  If it does not exist, the launcher defaults to C:\MentorGraphics\BridgePoint\license\license.dat .  Therefore, users of floating licenses must edit the launcher batch file to point to the server machine using the following notation:

<port_number>@<server_name>

Both port_number and server_name should match what is defined on the SERVER line in the license.dat file on the server (for example, 1717@svr-inl-eng-01 in our example).

You can now invoke BridgePoint from the client machines. They will automatically check out licenses (up to a designated maximum) from the server machine.

Adding New Products to an Existing License Server

If you have a pre-existing FLEXlm license server already running and you need to add new licensed products, you must do the following:

1.     Obtain license keys from Mentor Graphics for your new products.

2.     Stop the lmgrd process running on your server machine using the lmtools.exe configuration tool’s Start/Stop/Reread tab.

3.     Copy the license keys into your existing license.dat file (it may be named differently for other installations, so check the LM_LICENSE_FILE environment variable to be sure).

4.     There can only be one SERVER line in the license.dat file, so either comment out the newly-added SERVER line or the existing old one.

5.     Modify the newly added DAEMON lines to point to the path to the appropriate vendor daemons.

6.     Restart lmgrd.

NOTE: If you have several versions of the FLEXlm utility programs, use the latest version of lmgrd.  Version information can be obtained by using <license_manager_dir>\lmutil lmver lmgrd.

Dongle-Based Licenses

A "dongle" is a hardware key that plugs into your computer (either a USB or Parallel Port).

If you were issued a Licensing Dongle:

1.     Install the FLEXid driver for the dongle if it is not already installed.  Find the driver on the BridgePoint Distribution CD image in <CD-ROM>\license\Dongle.

2.     Attach the dongle to your PC (either to a USB or parallel port, depending on what you were issued).

3.     Verify that the dongle is active by issuing the following command at a command prompt:

<license_manager_dir>\license\lmutil lmhostid -flexid

A message similar to the following informs you of the dongle's ID number:

The FLEXlm host ID of this machine is "<hostnumber>"

If host ID number does not match the ID listed in the license.dat file, then you will need to request a corrected license with a matching host ID.  Find the license.dat file in the license directory under your BridgePoint installation.  Typically, the location is C:\MentorGraphics\BridgePoint\license\license.dat .

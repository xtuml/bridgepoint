Class Diagram

A Subsystem Package is used to document classes and associations.  It may contain these elements: class, imported class, association, associative link, supertype, and subtype.  It may also contain nested subsystem packages or any of these diagram packages; activity, communication, sequence, use case

Class

   A class abstracts the common characteristics of a set of instances that share the same attributes, operations, associations and behavior.

Graphical Notation

A class is represented by a rectangle with three compartments. 

Name Compartment

The name compartment (top) shows the name of the class centered in bold.   Two properties of the class, class number and class key letter (in that order), are shown in the bottom right-hand corner of the name compartment surrounded by curly braces.  Every class has a name, a number, and a key letter.

Attribute Compartment

The attribute compartment shows an enumerated list of the attributes of the class.  Attributes are shown as <derived> <name> : <type> {<property-string>} where <derived> is “/” if the attribute is derived (not shown otherwise), <name> is the name of the attribute, and <type> is its type. 

Identifying attributes are indicated using a right justified <property-string> which carries the text I, I2, or I3 for primary, secondary, and ternary identifiers respectively.  Attributes created by the formalization of an association, are shown by a <property-string> with the text “R<relationship number>”, e.g., {R4}.   Combined formalizations are shown in sequence, e.g., {R1,R3}.  Constrained referential attributes are shown with a “c” following the <property-string>, e.g., {R4c}.

Multiple properties are comma delimited, identifiers followed by relationship numbers.  The property strings are enclosed in curly braces.

All parts that comprise the attribute name may be suppressed from the diagram except <name>. 

Operation Compartment

Operations are shown in the bottom compartment preceded by a stereotype indicating <<operation>>.

Operations are shown as <name> (<parameter-list>) : <type>, where <name> is the name of the operation (underlined if class-based), and <type> is the return type. 

<parameter_list> is a comma separated list of parameters specified as <name> : <type>, where <name> is the name of the parameter, and <type> is its type.

Imported Class

An imported class represents a class defined in another Subsystem Package (on another Class Diagram).  This is useful in representing associations that span subsystem packages.

Graphical Notation

An imported class is shown as a grey-colored class with the stereotype <<Imported>> shown in the top compartment.  The rest of the graphical representation is the same as for class.

Association 

An association abstracts a binary relationship between two classes.

Graphical Notation

An association is shown as a solid line from one class to itself (reflexive association) or to another class.  The association name is comprised of the literal “R” followed by the relationship number, e.g., R3, and is shown in the middle and above the line.  A slash (“/”) is shown in front of the relationship number if the association is derived (i.e., compositional) with the relationship numbers that compose the association shown in curly braces, separated by commas, e.g.,  /R3 {R1,R2}.

The following are shown at each association end, in the given order, starting at the class at the association end:

·         Multiplicity consisting of exactly one of the following:

o        1 (exactly one)

o        0..1 (zero or one)

o        1..* (one or more)

o        (zero or more)

·         Verb phrases or role names consisting of free-form text.

The relationship number and the verb phrases are moveable as desired by the user.  Multiplicity is fixed close to the association end and above the line.

Associative Link

An associative link represents the formalization of an association between two classes, in a class called an associative class. 

Graphical Notation

An associative link is a dashed line that starts at an associative class and ends at an association line of a binary association.  Associative classes look just like classes (i.e., the symbology is the same) and have the features and properties of classe s (attributes, operations, class number, class key letter, etc.), and can have associations with other classes.

When an association is formalized in an associative class, the association can have a multiplicity of one or many:  one in the case where only one instance of the association can exist between instances of the two associated classes, many when more than one instance of the association can exist.  This multiplicity is shown as a property string enclosed in curly braces near the intersection of the association line and the associative link, {1} for one (the default), and {*} for many.

Supertype/Subtype

Graphical Notation

A supertype/subtype association is shown as a solid line from the supertype class to the subtype class.  The hollow triangle is shown at the supertype end of the association.

The association is drawn by selecting the supertype button and drawing from the source to a position (waypoint) on the diagram. 

The user then selects the subtype button and finishes the association from the destination class to the waypoint.


The following adornments are shown:

·         The relationship number, e.g., R12, is placed near the hollow triangle.

·         The property string {disjoint, complete} is shown near the hollow triangle to indicate that supertype/subtype associations are mutually exclusive.

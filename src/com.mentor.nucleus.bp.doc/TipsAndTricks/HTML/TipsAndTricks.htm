Tips and Tricks

The following tips and tricks give some helpful ideas for increasing your productivity.

General Tips

Tip

Description

Working with the Sample Models

BridgePoint UML Suite ships with a sample microwave oven model.  The Getting Started section introduces you to the tool suite in the context of the sample models.

Importing

If you have existing BridgePoint 6.1 SQL files you may create new xtUML models from them using the new xtUML model wizard and specifying an import specification.

Files with an .sql extension

Files with an extension of .sql are pre-V7.0 xtUML model files.  If you don’t have BridgePoint version 6.1 installed, then you won’t have to worry about these files.

Using Local History to retrieve large model files

Eclipse can store previous versions of files in its Local History cache.  To retrieve Local History versions of a file, right-click on a file and select Replace with > Local History .. from the context menu.  To assure that large files such as BridgePoint UML Suite model (.xtuml) files are available from the Local History, make sure it is configured to store large files. To do this, from the menu bar, select Windows > Preferences. Then, select General > Workspace > Local History.  The maximum file size option should be set to at least 1 MB.

Diagram Tips

Feature

Description

Empty Diagrams

If your diagram is empty, it may be that the model elements are not currently in view. Press the "Show Entire Canvas" button on the diagram toolbar to place all of the model elements in view.

Panning a Diagram

To pan a diagram, hold down the spacebar and a hand will appear. Left-click and the hand will grab the diagram. Use the mouse to pan to the desired location.

Zooming in and out

The short cut keys for zoom in and zoom out are Ctrl-Shift-Up Arrow and Ctrl-Shift-Down Arrow respectively.

Creating and Deleting Waypoints

To create a waypoint on a connector, select it with a left-click and drag. To delete a waypoint, select the vertex and drag it back to its original position.

Compare Tips

Feature

Description

Which side is which?

The model text of the "newer" file is always shown on the left-hand side. Newer means the top-most file (when comparing two different files) or the truly more recent file when comparing two revisions of the same file.

The Overview Pane

The Overview pane shows compare icons relative to the left-hand side text. A + means a model element was added, a - means it was deleted, and no icon means that the model element was present in both models but was changed.

Narrowing the Compare

Double-clicking on a model element in the Overview pane restricts (or focuses) the compare to the selected model element.

Drilling down

Since model compare is textual, it can sometimes interpret changes incorrectly when evaluating all of the changes at the same time. To get a better idea of the changes, drill down using the Overview pane and double-click to see a comparison of the changed model element.

Introduction to BridgePoint UML Suite

Getting Started with the BridgePoint UML Suite

Welcome to the BridgePoint UML Suite!   This tool suite provides you with the tools necessary to get your applications up and running in minimal time with maximal quality.  By specifying your application in xtUML and translating into an implementation in either C or C++, you can go from models to code within minutes.

The Quick Tour presented here explores a sample model to demonstrate the key features of the BridgePoint UML Suite in a real-world application—that of a microwave oven.  As the name suggests, the tour is meant to be brief but also comprehensive, walking you through the entire tool suite from xtUML to code.  In the course of the tour, you will learn the fundamental entities of xtUML and get familiar with the mechanics of using the tool suite.  In particular, you will:

·         Review the sample xtUML model

·         Translate the sample models  to code

·         Model debug a pre-translated sample model in terms of the xtUML model elements

Sample Models

The sample model for the microwave oven, though simplified for the tour, implements the operational behavior of a typical microwave oven.  You open a door, put something inside to be heated, close the door, and enter the duration of time you want the oven to cook.  When the oven is finished cooking, a beeper sounds.  If the door is opened during cooking, the oven interrupts its operation.

A Few Terms Before We Get Started

The BridgePoint UML Suite is comprised of a set of plug-ins integrated into the Eclipse Development Environment.  For embedded developers that have not used Eclipse before, some terms and concepts may be unfamiliar.  Let’s take a look at these terms before moving on.

·         A project is the fundamental unit of organization in Eclipse. Most operations require the context of a project.

·         A workspace holds one or more related projects.

·         The workbench is the outermost container of an Eclipse-based user interface, which holds all the tools being used at a given time.

·         An individual tool panel is called a view.

·         Editors let you manipulate a resource directly, for example a source file, or model elements, such as attributes, activities, and descriptions.

·         A perspective contains a group of related views and editors. You can switch between perspectives to quickly access various task-centered groupings of functionality within the context of a single project or workspace.

The BridgePoint UML Suite builds upon the Eclipse fundamentals to provide projects, perspectives, views, and editors that let you build xtUML models and translate them into code.

The xtUML Modeling Perspective

When you view or enter a xtUML model, you do so from within the xtUML Modeling perspective.  As described above, a perspective is just a task-centered collection of views and editors.  The xtUML Modeling perspective provides views and editors that center around modeling in xtUML.   

As we go through the Quick Tour, you will be introduced to each of the views and editors that comprise the xtUML Modeling perspective.
